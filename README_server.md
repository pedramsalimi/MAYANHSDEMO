# Smart Mirror — Pose Comparison System

## Setup

### Requirements

Python 3.10 or higher. Install dependencies:

```
pip install -r requirements.txt
```

If `uvicorn` is not on your PATH after installing (common on Windows with conda):

```
python -m uvicorn realtime_server:app --host 0.0.0.0 --port 8000
```

### Running the server

```
uvicorn realtime_server:app --host 0.0.0.0 --port 8000
```

Then open `http://localhost:8000` in a browser.

> Use `http://localhost` not `http://127.0.0.1` — Chrome requires the former for webcam access.

Check the server is up:

```
curl http://localhost:8000/health
```

Interactive API docs (auto-generated by FastAPI):

```
http://localhost:8000/docs
```

## API

### POST /compare

Upload an instructor video and a patient video. The server processes both, compares every patient frame against the closest matching instructor frame using joint angle differences, and returns a JSON summary.

```bash
curl -X POST http://localhost:8000/compare \
  -F "ref=@instructor.mp4" \
  -F "trgt=@patient.mp4" \
  -F "model_name=yolov8m-pose.pt" \
  -F "offset=25"
```

**Parameters**

| Parameter | Required | Default | Description |
|---|---|---|---|
| `ref` | yes | — | Instructor video file |
| `trgt` | yes | — | Patient video file |
| `model_name` | no | `yolov8m-pose.pt` | YOLO model. Options: `yolov8n-pose.pt` (fast), `yolov8m-pose.pt` (balanced), `yolov8x-pose.pt` (accurate) |
| `offset` | no | `25` | Angle threshold in degrees. A joint is OK if the difference is below this value |
| `conf` | no | `0.5` | YOLO keypoint detection confidence threshold (0.0–1.0) |

**Response**

```json
{
  "meta": {
    "total_patient_frames":    1609,
    "total_instructor_frames": 1923,
    "ok_frames":               1124,
    "ok_pct":                  69.9,
    "angle_threshold":         25,
    "ref_detection_pct":       99.9,
    "trgt_detection_pct":      100.0
  },
  "joint_stats": {
    "right_shoulder": { "bad_frames": 48, "bad_pct": 14.9, "mean_diff": -8.2, "max_abs_diff": 52 },
    "right_arm":      { "bad_frames": 12, "bad_pct":  3.7, "mean_diff":  2.1, "max_abs_diff": 38 },
    "left_shoulder":  { ... },
    "left_arm":       { ... },
    "right_hip":      { ... },
    "right_leg":      { ... },
    "left_hip":       { ... },
    "left_leg":       { ... }
  },
  "frames": [
    {
      "patient_frame":    0,
      "instructor_frame": 412,
      "mirrored":         false,
      "all_ok":           true,
      "joints": {
        "right_shoulder": { "ref": 136, "trgt": 128, "diff":  8, "status": "OK"  },
        "right_arm":      { "ref":  33, "trgt":  71, "diff": -38, "status": "BAD" }
      }
    }
  ]
}
```

**Key fields**

- `ok_pct` — percentage of patient frames where all 8 joints were within the threshold
- `joint_stats[j].mean_diff` — positive means patient's joint is more open than instructor, negative means more closed
- `ref_detection_pct` — percentage of instructor frames where YOLO successfully detected a person. Frames where detection failed fall back to 0° for all joints
- `trgt_detection_pct` — same as above but for the patient video. If either value is below ~80%, results may be unreliable — check that the full body is in frame and lighting is adequate
- `frames[].patient_frame` — index of the patient frame (sampled every 5th frame, so values are 0, 5, 10, 15, ...)
- `frames[].instructor_frame` — which instructor frame was the closest match for that patient frame
- `frames[].mirrored` — true if the patient turned in the opposite direction to the instructor (left/right joints were swapped before comparing)
- `frames[].joints[j].diff` — instructor angle minus patient angle

## Troubleshooting

**`uvicorn not recognised`**
```
pip install "uvicorn[standard]"
```

**`No module named scipy`**
```
pip install scipy
```

**`WS closed (code 1006)`** — WebSocket library missing:
```
pip install "uvicorn[standard]" websockets
```

**No person detected** — stand further back so full body is visible in frame, or lower `conf` to `0.3`.

**Camera access denied on Windows** — `Win+I → Privacy & Security → Camera → On`. Also check no other app (Teams, Zoom, OBS) is using the camera.

**Form accuracy is 0%** — the angle threshold may be too tight. Try `offset=35` or `offset=40`.
